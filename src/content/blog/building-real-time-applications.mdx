# Building Real-Time Applications with Socket.io

Real-time features have become essential in modern web applications. From live chat to real-time notifications, users expect instant updates. Socket.io makes this surprisingly easy.

## What is Socket.io?

Socket.io is a JavaScript library that enables real-time, bidirectional communication between web clients and servers. It abstracts the WebSocket API and provides additional features like auto-reconnection and fallback transport.

## Getting Started

### Server Setup

```javascript
const express = require('express');
const http = require('http');
const { Server } = require('socket.io');

const app = express();
const server = http.createServer(app);
const io = new Server(server, {
  cors: {
    origin: "*"
  }
});

io.on('connection', (socket) => {
  console.log('User connected:', socket.id);
  
  socket.on('disconnect', () => {
    console.log('User disconnected:', socket.id);
  });
});

server.listen(3000, () => {
  console.log('Server running on port 3000');
});
```

### Client Setup

```javascript
import { io } from 'socket.io-client';

const socket = io('http://localhost:3000');

socket.on('connect', () => {
  console.log('Connected to server');
});
```

## Real-Time Notifications

A common use case is broadcasting notifications to all connected clients:

### Server

```javascript
io.on('connection', (socket) => {
  socket.on('send-notification', (data) => {
    io.emit('new-notification', {
      message: data.message,
      timestamp: new Date()
    });
  });
});
```

### Client

```javascript
socket.on('new-notification', (data) => {
  showToast(data.message);
});
```

## Room-Based Communication

Socket.io supports "rooms" for broadcasting to specific groups:

```javascript
// Join a room
socket.join(`user-${userId}`);

// Send to specific user
io.to(`user-${userId}`).emit('notification', data);

// Leave a room
socket.leave(`user-${userId}`);
```

## Practical Example: Order Tracking

Here's how I implemented real-time order tracking in the Pizza Delivery app:

### Server

```javascript
io.on('connection', (socket) => {
  socket.on('track-order', (orderId) => {
    socket.join(`order-${orderId}`);
  });
});

function updateOrderStatus(orderId, status) {
  io.to(`order-${orderId}`).emit('status-update', {
    orderId,
    status,
    timestamp: new Date()
  });
}
```

### Client

```javascript
socket.emit('track-order', orderId);

socket.on('status-update', (data) => {
  updateOrderUI(data.status);
  showNotification(`Order is now: ${data.status}`);
});
```

## Best Practices

### 1. Handle Disconnections
Always handle client disconnections gracefully:

```javascript
socket.on('disconnect', () => {
  // Clean up user-specific data
  cleanupUserData(socket.id);
});
```

### 2. Authentication
Authenticate socket connections:

```javascript
io.use((socket, next) => {
  const token = socket.handshake.auth.token;
  if (isValidToken(token)) {
    next();
  } else {
    next(new Error('Authentication error'));
  }
});
```

### 3. Rate Limiting
Prevent abuse by implementing rate limits:

```javascript
const rateLimit = require('socket.io-rate-limit');

io.use(rateLimit({
  windowMs: 1000,
  max: 10
}));
```

### 4. Error Handling
Always wrap socket handlers in try-catch:

```javascript
socket.on('action', async (data) => {
  try {
    const result = await processAction(data);
    socket.emit('action-complete', result);
  } catch (error) {
    socket.emit('error', { message: error.message });
  }
});
```

## Scaling Considerations

For production, you'll need to use Redis or another adapter to handle multiple server instances:

```javascript
const { createAdapter } = require('@socket.io/redis-adapter');
const { createClient } = require('redis');

const pubClient = createClient({ url: 'redis://localhost:6379' });
const subClient = pubClient.duplicate();

await Promise.all([pubClient.connect(), subClient.connect()]);

io.adapter(createAdapter(pubClient, subClient));
```
